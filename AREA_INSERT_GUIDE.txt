===============================================
AREA TABLE - DATA INSERTION GUIDE
===============================================

CSV File: Area_Data.csv
Total Records: 135 Areas
Database: waste_management
Table: Area

COLUMNS:
1. Area_ID (VARCHAR 10) - Primary Key (A-101 to A-235)
2. Area_Code (VARCHAR 20) - Unique Code (DNCC-01 to DNCC-100, DSCC-01 to DSCC-35)
3. Area_Name (VARCHAR 100) - Area Name/Location
4. Population (INT) - Population count (currently empty)
5. Supervisor (VARCHAR 100) - Supervisor name (currently empty)
6. Description (TEXT) - Additional description (currently empty)

CSV STRUCTURE:
- Format: Comma-separated values
- Header: Area_ID,Area_Code,Area_Name,Population,Supervisor,Description
- Records: 135 data rows
- Coverage: 100 DNCC zones + 35 DSCC zones in Dhaka

GEOGRAPHY:
- DNCC (Dhaka North City Corporation): 100 areas (A-101 to A-200)
- DSCC (Dhaka South City Corporation): 35 areas (A-201 to A-235)

SAMPLE RECORDS:
A-101,DNCC-01,Gulshan 1,,,
A-102,DNCC-02,Gulshan 2,,,
...
A-235,DSCC-35,Islampur,,,

SQL INSERT EXAMPLE (Manual):
INSERT INTO Area (Area_ID, Area_Code, Area_Name) 
VALUES ('A-101', 'DNCC-01', 'Gulshan 1');

MYSQL LOAD INFILE (Bulk Import):
LOAD DATA LOCAL INFILE '/path/to/Area_Data.csv'
INTO TABLE Area
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n'
IGNORE 1 ROWS
(Area_ID, Area_Code, Area_Name, Population, Supervisor, Description);

NOTES:
- All Area_IDs are unique and form a complete sequential set
- Area_Codes follow DNCC/DSCC naming convention
- Population, Supervisor, and Description fields are currently empty
- You can update these fields later with actual data
- All records are already in the database and ready for export

SOURCE: waste_management database (XAMPP MySQL)
